# User Interface Requirements

This document describes **what** the UI must do, at spec fidelity. Exact visual design (color palettes, typography) is deferred to implementation.

## Screens

| Screen | Purpose |
|---|---|
| Dashboard | Overview + expiring/low-stock widgets + quick-add entry point. |
| Inventory list | Primary list view, filterable, sortable. |
| Item detail | Full view of a single item, including photos and quantity history. |
| Quick-add modal | Fast creation/increment flow. Reachable from everywhere. |
| Search | Dedicated search surface with filters. |
| Floor plan (View / Edit) | Visual navigation; edit mode toggles in. |
| Shopping list | Auto + manual entries, restock flow. |
| Settings | Household-wide settings, including Item Types, Locations, Backups, Expiring-soon window. |

## Global requirements

- **FR-UI-001**: A persistent navigation element must expose Dashboard, Inventory list, Search, Floor plan, Shopping list, and Settings from any screen. On mobile this is a bottom tab bar; on desktop a left sidebar.
- **FR-UI-002**: A **Quick-add** button must be visible on every primary screen, regardless of viewport width, and must be reachable with a single tap or a single keyboard shortcut (`q` or `n`).
- **FR-UI-003**: The app must be responsive across the viewports in `NFR-USA-001`.
- **FR-UI-004**: All forms must show inline validation errors next to the offending field; the form must not submit when errors are present.
- **FR-UI-005**: Destructive actions (delete item, delete type, delete location, restore from backup) must require confirmation. Restore requires double-confirmation per `FR-BACKUP-031`.

## Dashboard

- **FR-UI-010**: The Dashboard must show, in order:
  1. Greeting area / app name.
  2. Alerts section: **Expired** widget, **Expiring soon** widget, **Low stock** widget (each collapses to a single-line confirmation when empty).
  3. Recent additions (last 10 created items).
  4. Quick-add call-to-action (redundant with the global button, emphasized here).
- **FR-UI-011**: Each alert widget must link to the relevant filtered Inventory list.

## Inventory list

- **FR-UI-020**: The list shows items in rows with the fields defined in `FR-SEARCH-030`.
- **FR-UI-021**: Filter chips reflect any active filter and can be dismissed individually.
- **FR-UI-022**: A sort selector offers the sorts listed in `FR-SEARCH-004`.
- **FR-UI-023**: Pagination controls must show current page and total pages and support keyboard navigation.
- **FR-UI-024**: Empty-state text guides the user toward Quick-add or clearing filters (per `FR-SEARCH-040`).

## Item detail

- **FR-UI-030**: Must render all item fields (`03-domain-model.md#item`), including the inherited-from-type indicator described in `FR-STOCK-009`.
- **FR-UI-031**: Must include: inline quantity `-`/`+`/`set` controls, edit button, delete button with confirmation, and an activity feed of the last 10 `QuantityChange` records.
- **FR-UI-032**: Photo gallery must support the reorder affordance described in `FR-PHOTOS-012`.

## Quick-add modal

- **FR-UI-040**: Opens on top of the current screen; closing returns the user to exactly where they were (no scroll loss).
- **FR-UI-041**: Enforces the **≤ 3 interaction** budget of `NFR-PERF-002`:
  1. Open (button or shortcut).
  2. Type / select name (autocomplete active).
  3. Save (Enter or button).
- **FR-UI-042**: Displays defaults for type / location / unit inline beneath the name field with "change" affordances that do not count against the 3-interaction budget if unused.
- **FR-UI-043**: On successful save, the modal closes and a toast confirms the action (new vs. incremented).

## Search

- **FR-UI-050**: Text input is focused on screen open.
- **FR-UI-051**: Filter UI (chips + filter sheet/side panel) matches `FR-SEARCH-002`.
- **FR-UI-052**: Typing is debounced at 120 ms before a query fires.

## Floor plan

### View mode

- **FR-UI-060**: Renders the single `FloorPlan` using `Room.shape_on_plan` and `StorageLocation.shape_on_plan`.
- **FR-UI-061**: Supports pan and zoom per `FR-PLAN-002`.
- **FR-UI-062**: Location labels must meet `NFR-USA-006` contrast at all zoom levels; small shapes elide labels to fit (`FR-PLAN-005`).
- **FR-UI-063**: Tap/click targets on storage locations must meet `NFR-USA-002`.
- **FR-UI-064**: Badges render per `FR-PLAN-006`.

### Edit mode

- **FR-UI-070**: A prominent toggle switches to edit mode; edit-mode UI is visually distinct.
- **FR-UI-071**: Selection handles appear on shapes when they are selected in edit mode.
- **FR-UI-072**: Snap-to-grid default is enabled (`FR-PLAN-020`); toggle disables.
- **FR-UI-073**: Save and Discard actions are always visible in edit mode.
- **FR-UI-074**: Validation errors on save must visually highlight each offending shape with an accessible error message.

## Shopping list

- **FR-UI-080**: Matches the behavior in `04-functional/shopping-list.md`.
- **FR-UI-081**: Manual-entry input is pinned at the top for fast addition.
- **FR-UI-082**: Restock prompts for expired items open as bottom sheets on mobile and modals on desktop.

## Settings

- **FR-UI-090**: Settings groups: Item Types, Locations (redirect to the Locations screen), Expiring-soon window, Backups, System info.
- **FR-UI-091**: System info shows server address, server version, storage usage, last backup status.

## Accessibility and keyboard

- **FR-UI-100**: Tab order matches visual order; no keyboard traps.
- **FR-UI-101**: Esc closes modals and bottom sheets.
- **FR-UI-102**: `q` or `n` opens Quick-add from any screen (overridable per `NFR-I18N-002` in the future).
- **FR-UI-103**: All icons have `aria-label`s.
- **FR-UI-104**: Form fields have associated `<label>` elements.

## Errors

- **FR-UI-110**: Transient errors (network, validation) are shown inline.
- **FR-UI-111**: Unexpected errors are shown in a non-blocking toast with a "Report" affordance that copies context (URL, timestamp, error code) to the clipboard for easy logging by the user.
- **FR-UI-112**: A persistent banner is shown when the last automatic backup failed (`FR-BACKUP-006`).
